# EMS Dashboard — Revamp v2 Release Notes

**Release tag:** `dashboard@2.0.0`
**Ship date:** 2026-05-12
**Gate task:** REVAMP-114 (SHIP)
**Track:** UI Revamp v2 (REVAMP-01 → REVAMP-114, 114 tasks)

---

## TL;DR

Every screen in the school dashboard now speaks the same Superhuman/Linear-
style design language pioneered on the Staffs module. Single token set,
single primitive library, single shell. Dark mode, density toggle, `en` +
`hi` locales, and a full visual-regression baseline ship in this release.

There is no breaking API change. Backend, parent app, and staff app are
untouched. Plan-based feature gates and multi-tenant `schoolId` scoping
behave identically.

---

## What shipped

### Foundation
- **Tokens + classes** — `src/styles/tokens.css`, `staff.css`, `create.css`,
  `shell.css`, `frosted-overlay.css`. Inline hex codes and arbitrary
  Tailwind values (`w-[173px]`) are gone.
- **Primitives** — every interactive control consolidated to one
  implementation in `src/components/ui/`. Documented at the in-app
  `/style-guide` route.
- **Shell** — sidebar, topbar, command palette, and global feedback
  primitives rebuilt on the new tokens.

### Screens
| Module | Surfaces revamped |
| --- | --- |
| Auth | Login, Signup, Reset, Public Form |
| Dashboard | Home, Analytics, AI Assistant |
| Students | List, Add composer, Detail + overlay, Promotion + TC, modals |
| Staff | Dashboard, Payroll, secondary pages (legacy AddStaff retired) |
| Classes | List, Detail, Attendance, Timetable, modals |

### Cross-cutting
- Density / theme / locale toggles in topbar.
- `en` + `hi` translations across every revamped surface.
- RTL pass for Arabic-shaped layouts.
- Print stylesheets for receipts, ID cards, report cards, admit cards.
- Recharts + map themes wired to revamp tokens.
- Server-side PDF templates aligned with the new look.

### QA
- Visual regression baseline at `tests/visual/revamp.spec.ts`
  (Playwright × 3 breakpoints).
- QA matrix at `docs/QA_MATRIX.md` (theme × breakpoint × locale per
  screen).
- Lighthouse pass on the five hottest pages.

## Smoke-test matrix

Run with a seeded admin account on staging. All flows must complete
without console errors, with all four states (skeleton → success or
empty / error) rendering correctly.

| # | Flow | Steps | Pass criteria |
| - | --- | --- | --- |
| 1 | Login | `/login` → admin creds → land on `/dashboard` | JWT cookie set, sidebar populated |
| 2 | Create student | `/students` → "Add" → composer all sections → submit | Student appears in list with admission no. |
| 3 | Create staff | `/staff` → "Add" → composer all sections → submit | Staff appears in list, payroll defaulted |
| 4 | Mark attendance | `/classes/:id/attendance` → today → toggle present/absent → save | Persists on reload, stats update |
| 5 | Record payment | `/fees` → student → record payment → submit | Receipt rendered, balance updated |
| 6 | Send announcement | `/messaging/announcements` → new → audience + body → send | Appears in list, notification fan-out |
| 7 | View report | `/academics/exams/:id/results` → publish → student detail | Result card renders with grade/rank |
| 8 | Switch theme | Topbar → theme toggle | Tokens flip, no flash, no contrast loss |
| 9 | Switch locale | Topbar → locale → `hi` | All strings translated, no overflow |
| 10 | Logout | Topbar → user menu → logout | Cookie cleared, redirect to `/login` |

Smoke tests run by the operator on the staging deploy before flipping
the production DNS. Record pass/fail in the QA matrix's bug log section.
